# KurosioAuctionSystem
Minecraftサーバー内でオークションができるプラグイン


## 使い方 / コマンド集
出品したいアイテムを持ち、  
 ``/kac start <開始価格> [最低入札単位] [半径] ``  

でオークションをスタート。  
指定する半径以内のプレイヤーにオークションの開始とID等が表示されます。  
<デフォルト>  
最低入札単位：1000円  
半径：15ｍ（最大30ｍ）  

``/kac join <ID>``  ー指定IDのオークションに参加  

``/kac bid``　ーオークション参加後、最低入札単位分 入札  

``/kac bid <金額>``　ー指定金額分入札  

``/kac autobid <金額>``　ー指定金額分まで自動入札  

``/kac leave``　ーオークションから離脱 （最高入札者の場合落札者となります。） 

``/kac exlist``　ー参加・出品中のオークション情報を表示  

``/kac cancel``  ーオークションを中止 (出品者のみ)    

``/kac help``　　ーコマンド集を表示  

## プラグイン詳細
オークション開始後、出品者のアイテムはサーバーのメモリ上へ預かる形となります。  
``kac-auctions.yml``にも保存されるため二重管理となっています。  
また、アイテムを落札者に送信または出品者に返却される際、インベントリが満杯の場合、該当ユーザー付近の床に落とされるようになっています。


### kac-auctions.yml
オークション出品中の情報は ``kac-auctions.yml`` に保存されます。  
サーバーがオークション実行中にダウンした場合、オークションは中止され、   
復旧後、出品者がサーバーにログイン次第アイテムは返却されます。  
### returns.yml  
オークションが異常終了(サーバー再起動や出品者のサーバー移動等)した際に一時的に出品物を``returns.yml``に保存します。  
出品者がサーバーに再度入った時にアイテムを返却します。
### history.yml
オークションの結果が保存されます。  

## 自動入札のしくみ  
KACには自動入札機能があります。  
自動入札のしくみはこちらで記載されているものと同様となります。  
https://help.kankocho.jp/knowledge/help023  
(官公庁オークション)  
※入札単位はユーザーで設定していただけます。  

## 入札単位について  
入札単位はデフォルトで1000円ですが、1000円以外も設定が可能です。  
ただし、入札時の判定方法が、「現在の入札価格+入札単位の倍数分かどうか」なため、  
"10の乗数"に設定することをおすすめします。(例:100,1000,10000)
