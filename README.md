# KurosioAuctionSystem
Minecraftサーバー内でオークションができるプラグイン

## 使い方 / コマンド集
出品したいアイテムを持ち、  
 ``/kac start <開始価格> [u:<入札単位>] [r:<半径>] [auto:<on/off>] [exp:<MCID>] ``  

でオークションをスタート。  
指定する半径以内のプレイヤーにオークションの開始とID等が表示されます。  
また、onにすることで自動入札を使えるように設定します。  
exp:< MCID >を指定するとそのプレイヤーは参加・入札・自動入札できません。  
<デフォルト>  
最低入札単位：1000円  
半径：30ｍ（最大50ｍ）  
自動入札機能：off  

``/kac join <ID>``: 指定IDのオークションに参加  

``/kac bid``: オークション参加後、最低入札単位分 入札  

``/kac bid <金額>``: 指定金額分入札  

``/kac autobid <金額>``: 指定金額分まで自動入札  

``/kac leave``: オークションから離脱 （最高入札者の場合落札者となります。） 

``/kac exlist``: 参加・出品中のオークション情報を表示  

``/kac cancel``: オークションを中止 (出品者のみ)    

``/kac help``: コマンド集を表示  

## プラグイン詳細
オークション開始後、出品者のアイテムはサーバーのメモリ上へ預かる形となります。  
``kac-auctions.yml``にも保存されるため二重管理となっています。  
アイテムを落札者に送信、または出品者に返却される際、インベントリが満杯の場合は、[ItemStash](https://github.com/AzisabaNetwork/ItemStash) プラグインが導入されていればStashに保管されます。未導入の場合は対象プレイヤーの足元にドロップされます。

### kac-auctions.yml
オークション開催中の情報は ``kac-auctions.yml`` に保存されます。  
サーバーがオークション実行中にダウンしたりリロードされた場合、復旧後にオークションが再開されます。サーバーダウン中に終了時刻を迎えていた場合は、正常に終了処理（出品者への入金と落札者へのアイテム付与）が行われます。

### returns.yml  
オークション終了時に落札者がオフラインだった場合や、異常終了(サーバー再起動や出品者のサーバー移動等)した際に、出品物や落札アイテムを一時的に``returns.yml``に保存します。  
対象プレイヤーがサーバーに再度ログインした際にアイテムが自動的に付与されます。
### history.yml
オークションの結果が保存されます。  

## 自動入札のしくみ  
KACには自動入札機能があります。  
自動入札のしくみはこちらで記載されているものと同様となります。  
https://help.kankocho.jp/knowledge/help023  
(官公庁オークション)  
※入札単位はユーザーで設定していただけます。  

## 入札単位について  
入札単位はデフォルトで1000円ですが、1000円以外も設定が可能です。  
ただし、入札時の判定方法が、「現在の入札価格+入札単位の倍数分かどうか」なため、  
"10の乗数"に設定することをおすすめします。(例:100,1000,10000)

### 動作環境
- Spigot/Paper 1.15.2+
- Java 8+
- Vault
- Vault対応経済プラグイン

### インストール
1. VaultとVault対応の経済プラグインを導入
2. ItemStashを導入 (任意: アイテムロスト防止)
3. `KurosioAuctionSystem-x.x.x.jar` (またはビルド済みjar) を plugins フォルダへ配置
4. サーバーを起動

### ビルド方法 (開発者向け)
このプロジェクトはGradle (Kotlin DSL) を使用しています。
```bash
./gradlew build
```
ビルドが完了すると、`build/libs/` 内に `KurosioAuctionSystem-x.x.x.jar` が生成されます。
